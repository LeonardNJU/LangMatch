# 2048 重跑矩阵结果（中文分析版）

> **2048 重跑说明**
>
> 本文档对应的是统一输出预算 `2048` 之后的修正版矩阵结果。它修复了原始 pilot 中最严重的截断问题，尤其是 `gpt-5.4 / IFEval` 的异常低分。
>
> 重跑后的截断审计结果如下：
>
> - `gpt-4o`：`0` cap-hit，`0` 空响应
> - `gpt-5.4`：`0` cap-hit，`0` 空响应
> - `qwen3-4b`：`0` cap-hit，`0` 空响应
> - `qwen3-1.7b`：仍有 `4` 条 cap-hit（`finish_reason=length`），因此这个模型还保留少量残余截断风险
>
> 因而，这一版结果已经可以作为闭源模型与 `qwen3-4b` 的主比较版本；只有 `qwen3-1.7b` 仍需保留轻度方法学保留意见。

本文档给出 `2048` 重跑后的中文解读。评测同时比较了闭源模型与开源模型、三种提示模式，以及三个 benchmark，并同时保留任务分数与 token 成本，以呈现真实的质量—成本 tradeoff。

下方图使用 Pareto 风格散点图：

- 横轴：`Total Tok`（越低越好）
- 纵轴：`Score / SR`（越高越好）
- 颜色：模型
- 点形：prompt 模式

### 总体中文解读

- 这次重跑最重要的变化是：**`gpt-5.4` 变成了整体最强模型**，这说明原来它在 `IFEval` 上看起来偏弱，确实是 token cap 造成的假象。
- `zh_compact` 仍然是最稳的默认方案：它在 `gpt-4o` 上最好，在 `qwen3-4b` 上与 `wy` 打平，但 token 效率更高。
- `wy` 在强模型上的价值更加可信了：它现在不仅是 `gpt-5.4` overall 最优，也是 `gpt-5.4 / MATH-500` 最优。
- `qwen3-1.7b` 虽然比原 pilot 更可信，但仍然有 4 条 residual truncation，因此这一路结果要比其他三路更谨慎地解释。

## 2048 修正版的跨 benchmark 总结

### 1. 修正后最重要的变化：`gpt-5.4` 的表现被“翻正”了

在原 pilot 中，`gpt-5.4` 因为 IFEval 的 token cap 太紧而被严重低估。2048 重跑后，它不但恢复正常，而且成为总体最强模型。这个变化说明：**在 reasoning model 上，过紧的 completion cap 会直接扭曲结论。**

### 2. `zh_compact` 仍是最稳的默认值

即使在修正后版本中，`zh_compact` 仍然是跨模型最稳的 prompt 语言。它在 `gpt-4o` 和 `qwen3-4b` 上都表现最好或并列最好，而且在指令遵循场景里依然优势明显。

### 3. `wy` 的收益现在更可信，但仍然是条件性的

2048 重跑后，`wy` 在 `gpt-5.4` 上的优势不仅没消失，反而更可信了。尤其在 `MATH-500` 上，它依然是最优条件。这说明 Wenyan 的增益不是单纯幻觉，而是存在于“强模型 + 凝练推理任务”这一条件组合中。

### 4. 还剩一个残余问题：`qwen3-1.7b`

`qwen3-1.7b` 在 2048 条件下仍有 4 条 cap-hit，因此如果需要严格到论文定稿级别的公平性，这一路还应该继续提高 budget 再重跑一次。但它已经不再影响闭源模型与 `qwen3-4b` 的主结论。

## 建议写进正文的修正版结论

> 在统一输出预算为 2048 的修正矩阵中，文言文仍未表现为通用最优提示语言，但其效果在强模型上具有真实且可重复的条件性优势。现代中文压缩版是跨模型最稳的默认方案，而文言文在较强闭源模型、尤其是数学推理任务中，能够形成更优的质量—成本权衡。与此同时，较弱开源模型并不能稳定吸收文言压缩带来的潜在收益，说明“文言文作为自然语言 prompt compression”并不是普适技巧，而是一种显著依赖模型能力与任务结构的条件性策略。
